\chapter{Database Schema Definitions}\label{chap:database_schemas}
The following sections outline the detailed schema properties for the core system tables.

\textbf{User Profile Table}
Following the Phase 2 architectural consolidation, core identity profiles have been migrated to \textbf{Amazon DynamoDB} under the table name \texttt{phase2-merge-users}. This shift ensures that user profile data resides within the same high-availability cloud ecosystem as conversational state, facilitating unified data management. The profile information is critical as it provides the necessary context for the generative engine, allowing the system to tailor responses based on user-defined education levels and persona traits.

\chapter{API Specifications}\label{chap:api_specs}
The following sections provide comprehensive specifications for the Core API endpoints.

\subsection{Upload Files [\texttt{POST} \texttt{/api/upload}]}
Manages the initial ingestion of raw document assets into the system's storage backend (local filesystem or Amazon S3). This service supports high-concurrency multi-file uploads and ensures that all assets are logically partitioned by user identity. The corresponding user interface for document ingestion is illustrated in Figure \ref{fig:ui_upload}.

\begin{itemize}
    \item \textbf{Request Headers}:
    \begin{itemize}
        \item \texttt{X-User-Id} (String, Optional): The logical tenant ID used for workspace isolation and storage partitioning. Defaults to \texttt{default}.
    \end{itemize}
    \item \textbf{Request Body (multipart/form-data)}:
    \begin{itemize}
        \item \texttt{files} (Binary Array, Required): A collection of document assets to be streamed directly to persistent storage.
    \end{itemize}
    \item \textbf{Response}:
    \begin{itemize}
        \item \texttt{Response} (String): A confirmation message indicating the successful ingestion of the files.
    \end{itemize}
\end{itemize}

\subsection{Run Pipeline [\texttt{POST} \texttt{/api/process}]}
Initiates the 7-stage multimodal extraction and normalization pipeline for a specific set of assets with intelligent conditional routing. The visual orchestration of these pipeline stages is shown in Figure \ref{fig:ui_pipeline}. Key features:
\begin{itemize}
    \item \textbf{Stage 1}: Normalization and format detection - routes files to appropriate handlers
    \item \textbf{Stage 2}: Media processing with noise reduction (80Hz high-pass filter)
    \item \textbf{Stage 3}: Main document processing via Docling for non-specialized file types
    \item \textbf{Stage 3b} (conditional): Excel custom XML parser with table-aware chunking (max 2000 chars)
    \item \textbf{Stage 3c} (conditional): DOCX custom XML parser with heading-aware chunking
    \item \textbf{Stage 3d} (conditional): PDF custom parser with heading-aware chunking
    \item \textbf{Stage 4}: Consolidation merging all stage outputs into RAG-ready format
\end{itemize} 

\begin{itemize}
    \item \textbf{Request Headers}:
    \begin{itemize}
        \item \texttt{X-User-Id} (String, Optional): Identifier for multi-tenant workspace isolation.
    \end{itemize}
    \item \textbf{Query Parameters}:
    \begin{itemize}
        \item \texttt{force} (Boolean, Default: \texttt{false}): Toggles re-processing of already processed structural artifacts.
    \end{itemize}
    \item \textbf{Request Body}:
    \begin{itemize}
        \item \texttt{selected\_paths} (String Array, Optional): Specific file paths within the tenant workspace to process. 
        \item \texttt{mode} (String, Optional): Pipeline fidelity level (\texttt{standard} or \texttt{fast}).
    \end{itemize}
    \item \textbf{Response}:
    \begin{itemize}
        \item \texttt{status} (String): Overall pipeline state (\texttt{completed} or \texttt{failed}).
        \item \texttt{results} (Object): Granular execution statistics for each multimodal stage.
    \end{itemize}
\end{itemize}

\subsection{Build Index [\texttt{POST} \texttt{/api/index}]}
Synchronizes extracted artifacts into searchable vector and keyword indexes. This service manages both the Qdrant visual/text collections and the BM25 sparse index. The indexing management dashboard is presented in Figure \ref{fig:ui_index}.

\begin{itemize}
    \item \textbf{Request Headers}:
    \begin{itemize}
        \item \texttt{X-User-Id} (String, Optional): Identifier for multi-tenant index isolation.
    \end{itemize}
    \item \textbf{Query Parameters}:
    \begin{itemize}
        \item \texttt{force} (Boolean, Default: \texttt{false}): If \texttt{true}, forces re-indexing of artifacts even if they exist in the vector store.
    \end{itemize}
    \item \textbf{Request Body}:
    \begin{itemize}
        \item \texttt{selected\_paths} (String Array, Optional): Absolute paths to specific normalized artifacts for indexing.
        \item \texttt{selected\_names} (String Array, Optional): Match artifacts by original document filenames. 
        \item \texttt{mode} (String, Optional): Indexing modality (\texttt{standard} or \texttt{fast}).
    \end{itemize}
    \item \textbf{Response}:
    \begin{itemize}
        \item \texttt{status} (String): The final state of the indexing lifecycle.
        \item \texttt{results} (Object): Success metrics for text (dense/\textbf{BM25}) and visual indexing stages.
    \end{itemize}
\end{itemize}

\subsection{Knowledge Explorer [\texttt{POST} \texttt{/api/search}]}
Provides a high-fidelity search interface for granular document exploration, supporting hybrid retrieval (\textbf{BM25} + Dense) and visual context rendering using multimodal embeddings. A visual representation of the search interface is available in Figure \ref{fig:ui_explorer}.

\begin{itemize}
    \item \textbf{Request Headers}:
    \begin{itemize}
        \item \texttt{X-User-Id} (String, Optional): Multi-tenant identifier for workspace isolation and search auditing.
    \end{itemize}
    \item \textbf{Request Body}:
    \begin{itemize}
        \item \texttt{query} (String, Required): The natural language query from the end-user.
        \item \texttt{top\_k} (Integer, Optional): Number of candidate chunks to retrieve for ranking.
        \item \texttt{retriever\_type} (String, Optional): Retrieval logic (\texttt{bm25}, \texttt{dense}, or \texttt{hybrid}).
        \item \texttt{include\_images} (Boolean, Optional): Toggles the use of visual tokens in search.
        \item \texttt{images\_for\_generation} (Integer, Optional): Number of visual contexts to provide to the generator.
        \item \texttt{mode} (String, Optional): Operation modality (\texttt{retrieval\_only} vs \texttt{retrieval\_generation}).
        \item \texttt{search\_scope} (String, Optional): Target index (\texttt{text}, \texttt{image}, or \texttt{both}).
        \item \texttt{generation\_model} (String, Optional): Override for the default inference model ID.
        \item \texttt{skip\_reranker} (Boolean, Default: \texttt{true}): Globally disabled by default for latency optimization. Set to \texttt{false} to enable reranking with \texttt{bge-reranker-large}.
    \end{itemize}
    \item \textbf{Response}:
    \begin{itemize}
        \item \texttt{generation} (Object): The synthesized answer and associated model metadata.
        \item \texttt{telemetry} (Object): Performance metrics for retrieval and generation phases.
    \end{itemize}
\end{itemize}

\begin{itemize}

    \end{itemize}
    \item \textbf{Request Body}:
    \begin{itemize}
        \item \texttt{document\_id} (String, Required): Unique identifier for the source document context.

        \item \texttt{tone} (String, Optional): Stylistic orientation (\texttt{neutral}, \texttt{formal}, or \texttt{friendly}).

    \end{itemize}
\end{itemize}

\subsection{Lecture Viewer [\texttt{GET} \texttt{/api/processed-file}]}
Provides a secure mechanism to download or inline-preview single file artifacts (e.g., high-resolution Markdown, page images, or original assets) located within the multi-tenant processing tree. The integrated viewer interface for multimodal artifacts is shown in Figure \ref{fig:ui_viewer}. The service is strictly tenant-safe; it rejects all path traversal attempts and ensures that S3 keys remain confined to the user-specific processing prefix.

\begin{itemize}
    \item \textbf{Request Headers}:
    \begin{itemize}
        \item \texttt{X-User-Id} (String, Optional): Identifier for multi-tenant path isolation and S3 key prefixing.
    \end{itemize}
    \item \textbf{Query Parameters}:
    \begin{itemize}
        \item \texttt{rel\_path} (String, Required): Forward-slashed relative path within the tenant workspace (e.g., \texttt{stage4/doc.md}).
    \end{itemize}
    \item \textbf{Security Constraints}: 
    \begin{itemize}
        \item \textbf{Traversal Prevention}: Programmatic rejection of \texttt{..} sequences to prevent directory escape.
        \item \textbf{Tenant Lockdown}: Forced prefixing with the user-specific storage root (S3 or Local).
    \end{itemize}
    \item \textbf{Response}:
    \begin{itemize}
        \item \texttt{File stream} (Binary): The raw requested asset delivered with appropriate MIME types for inline rendering.
    \end{itemize}
\end{itemize}

\subsection{Chat Assistant [\texttt{POST} \texttt{/api/chat/stream}]}
The primary RAG orchestration endpoint for streaming conversational interaction and long-term memory management. The conversational interface for the RAG assistant is detailed in Figure \ref{fig:ui_chat}. This service utilizes DynamoDB for session state and prompt injection for personalized personalization.

\begin{itemize}
    \item \textbf{Request Headers}:
    \begin{itemize}
        \item \texttt{X-User-Id} (String, Optional): Multi-tenant identifier for session isolation and storage access.
    \end{itemize}
    \item \textbf{Request Body}:
    \begin{itemize}
        \item \texttt{query} (String, Required): The natural language query or question from the end-user.
        \item \texttt{session\_id} (String, Required): Unique identifier for the chat session for memory continuity.
        \item \texttt{top\_k} (Integer, Default: \texttt{8}): Number of relevant document chunks to inject into the context.
        \item \texttt{persona} (String, Optional): Stylistic tone override (e.g., \texttt{professor}, \texttt{concise}).
        \item \texttt{education\_description} (String, Optional): Student background context for personalization.
    \end{itemize}
    \item \textbf{Response}:
    \begin{itemize}
        \item \texttt{response} (String): Server-Sent Events (SSE) stream of generated AI answer tokens.
    \end{itemize}
\end{itemize}

\textbf{Execution Workflow:}
\begin{enumerate}
    \item \textbf{State Validation}: Confirm user tenancy and session integrity.
    \item \textbf{Hybrid Search}: Parallel retrieval across text (dense vector + BM25) and visual (ColQwen) indexes.
    \item \textbf{Augmentation}: Context injection into the dynamic LLM prompt template.
    \item \textbf{Streaming}: Real-time response delivery via Server-Sent Events (SSE).
\end{enumerate}

        \item \texttt{suggested\_action} (String): AI-generated prescriptive improvement step.
        \item \texttt{analysis\_summary} (String): Narrative synthesis of the feedback generated by the LLM.
        \item \texttt{classifier\_model} (String): The ID of the model used for automated categorization.
        \item \texttt{classification\_status} (String): Pipeline state (\texttt{pending}, \texttt{completed}).
        \item \texttt{classification\_error} (String): Error trace if the AI analysis failed.
        \item \texttt{created\_at} (String): ISO-8601 timestamp of record creation.
        \item \texttt{updated\_at} (String): ISO-8601 timestamp of the last record update.
        \item \texttt{version} (Integer): Optimistic concurrency version for DynamoDB.
    \end{itemize}
\end{itemize}
