## Test Plan – Manual Testing for Zero Bank Project


### 1. Project Name:

Zero Bank Manual Testing


### 2. Objective / Purpose:

To ensure the core banking features of the Zero Bank website (http://zero.webappsecurity.com/) work as expected under valid and invalid user inputs, and to identify and report defects for resolution.


### 3. Scope of Testing:


#### In Scope:

- Login/Logout functionality
- Account summary viewing
- Transfer Funds
- Feedback submission


### 4. Testing Type:

- Manual Testing
- Functional Testing
- Smoke


### 5. Test Environment:

- Browser: Chrome v134+,
- OS: Windows 10
- Site URL: http://zero.webappsecurity.com


### 6. Test Deliverables:

- Test Plan
- Test Scenarios
- Test Cases
- RTM (Requirement Traceability Matrix)
- Defect Report / Bug Report
- Final Test Summary Report


### 7. Entry Criteria:

- Application is deployed and stable
- Requirements and functionalities are understood
- Test cases are reviewed


### 8. Exit Criteria:

- All critical test cases executed
- All major bugs resolved
- RTM is fully covered
- Test summary report submitted
