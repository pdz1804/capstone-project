# BK-MInD JMeter Main APIs Focus

This package isolates the most critical APIs into one focused suite:
- high-error search generation APIs,
- core mutation pipeline APIs (`upload`, `process`, `index`),
- focused dashboard/report output.

Folder: `docs/jmeter-main-apis`

## Why This Exists
From current standard battery results (`summary_labels_20260419_221347.csv`), the highest failures are concentrated on:
- `POST /api/search (retrieval_generation text)`
- `POST /api/search (retrieval_generation both)`

Also, previous run artifacts did not include:
- `POST /api/upload (optional)`
- `POST /api/process (optional)`
- `POST /api/index (optional)`

So this suite tests those APIs directly in one place.

## Retry Guidance (When Failures Are Transient)
Yes, in this stack retries can help.
Transient issues can happen on cloud dependencies (SageMaker, Bedrock, Qdrant cloud, network spikes).

Runner policy:
- bounded retries only (`-MaxRetries`, default 2),
- retry on failed run/JTL generation,
- no infinite loop.

Recommended operational practice:
- if repeated transient errors occur, rerun after a short cooldown and compare error labels in the next summary.

## Windows Stability Note (New)
During focused-suite debugging, native JVM crash `-1073741819` was observed intermittently when passing many runtime overrides directly via long CLI `-J...` chains.

Runner mitigation now applied:
- launch JMeter through `java -jar ApacheJMeter.jar` (headless),
- write runtime overrides to `reports/main_apis_focus_<timestamp>.runtime.properties`,
- pass that file with `-q` instead of a long `-J...` command line.

This keeps behavior reproducible and avoids brittle command-line parsing paths on Windows.

## Run Focused Test
```powershell
cd docs/jmeter-main-apis
Remove-Item Env:JVM_ARGS -ErrorAction SilentlyContinue
Remove-Item Env:HEAP -ErrorAction SilentlyContinue
.\run_main_apis_focus_tests.ps1
```

### Common options
```powershell
# Search-only focused run
.\run_main_apis_focus_tests.ps1 -RunSearchApis -RunPipelineApis:$false -Threads 10 -RampUp 60 -Loops 2

# Pipeline-only run (upload/process/index)
.\run_main_apis_focus_tests.ps1 -RunSearchApis:$false -RunPipelineApis -Threads 1 -Loops 1

# Keep existing users.csv and skip setup
.\run_main_apis_focus_tests.ps1 -SkipAccountSetup
```

## Dashboard Generation Notes (Aligned with JMeter 14.x)
This package follows JMeter dashboard rules:
- CSV save-service fields required for dashboard are written into runtime properties generated by runner.
- HTML dashboard is generated with `-e -o`.
- Focused series filtering is set in `main_apis_reportgenerator.properties` via:
  - `jmeter.reportgenerator.exporter.html.series_filter`
  - `jmeter.reportgenerator.exporter.html.filters_only_sample_series=true`
- APDEX and percentile properties are set (`90/95/99`, APDEX thresholds).

Manual generation options:

```powershell
# Generate from existing log
jmeter -g .\reports\main_apis_focus_<ts>.jtl -o .\reports\main_apis_focus_<ts>_regen

# Generate after test run
jmeter -n -t .\BKMind_Main_APIs_Focus.jmx -l .\reports\main_apis_focus_<ts>.jtl -e -o .\reports\main_apis_focus_<ts>
```

## Important Execution Detail
`/api/process` and `/api/index` use backend-side lock protection for concurrent jobs.
To avoid artificial 409 conflicts from test concurrency, pipeline APIs in this plan execute only on the first thread via IfController condition.
